Use this template when creating the local plan file during Phase 5 (Local Plan Storage).

```markdown
# Sprint: [Issue Key/ID] - [Summary]

**Work Item:** [Link to Jira issue or Linear Issue]
**Branch:** [branch-name]
**Created:** [date]

## Context

[Brief description from work tracker]

- [ ] [Criterion 2]
- [ ] [Criterion 3]

## Agreed Test Scope

_Sourced from Jira/Linear Sprint comment (generated by /aidlc-design). Empty if not available._

| Scenario | Layer | Priority | Notes |
|----------|-------|----------|-------|

## Constraints

- [Hard constraint from Rules section, if Task Spec format]

## Test Plan

### Unit Tests
- [ ] [Test 1]
- [ ] [Test 2]

### Integration Tests
- [ ] [Test 1]

## Testability Design Review

For every external dependency this sprint introduces (queue client, HTTP client, DB client,
clock/time, file system, randomness source), identify the seam that enables unit testing:

| Dependency | Seam type | Resolution |
|------------|-----------|------------|
| [e.g. ServiceBusClient] | [interface / factory / Func<T>] | [inject via ctor / integration test named X] |

> **Rule:** "Skip happy-path test" is not a permitted resolution. Choose one of:
> (a) extract a seam, (b) test at integration level (name the test), (c) refactor the call site.

## AC-to-Test Mapping

Every AC must map to at least one named test before plan approval. Rows with "TBD" are blockers.

| AC # | AC Summary | Test File :: Method | Assertion |
|------|-----------|---------------------|-----------|
| 1 | [AC text] | [TestClass.cs :: MethodName] | [what it asserts] |

## Contract Decisions

Document serialization, error, and connection contracts for every payload or DTO this sprint touches:

**Serialization:** [e.g. EventType enum → string via StringEnumConverter; null fields omitted; dates as ISO 8601]
**Error contract:** [e.g. DTO surfaces string ErrorMessage, never raw Exception type; stack trace stays in logs only]
**Connection lifecycle:** [e.g. CloseAsync in finally; failure logged at Warning; never swallowed silently]

## Dead-code Sweep Commitment

Before REFACTOR is marked complete on any cycle: every property on every new request/response
DTO will be diffed against the final serialised payload and its consumers. Unused properties will be removed.

## Implementation Progress

[Space for implementation notes]
```
